# AstroStorm - Future Development Ideas

This document outlines serious, implementable features and improvements for AstroStorm that can be developed locally without cloud services, databases, or authentication systems.

---

## 1. Additional Divisional Charts (Vargas)

Implement the complete set of 16 Shodashvarga charts:
- **D2 (Hora)** - Wealth and prosperity analysis
- **D3 (Drekkana)** - Siblings and courage
- **D4 (Chaturthamsa)** - Fortune, property, and vehicles
- **D7 (Saptamsa)** - Children and progeny
- **D12 (Dwadasamsa)** - Parents and ancestry
- **D16 (Shodasamsa)** - Vehicles, conveyances, and happiness
- **D20 (Vimsamsa)** - Spiritual progress and religious inclinations
- **D24 (Chaturvimsamsa)** - Education and learning
- **D27 (Bhamsa/Nakshatramsa)** - General strength analysis
- **D30 (Trimsamsa)** - Evils, misfortunes, and character
- **D40 (Khavedamsa)** - Auspicious/inauspicious effects
- **D45 (Akshavedamsa)** - All-round prosperity

## 2. South Indian Chart Style

Implement the traditional South Indian box chart format:
- Fixed house positions (signs rotate, houses stay fixed)
- Different visual representation preferred in South India
- User preference setting to switch between styles
- Proper handling of sign placement vs house placement

## 3. Shadbala (Six Strengths) Calculator

Implement precise Shadbala calculations:
- **Sthana Bala** - Positional strength (exaltation, own sign, etc.)
- **Dig Bala** - Directional strength
- **Kala Bala** - Temporal strength (hora, masa, varsha)
- **Chesta Bala** - Motional strength (retrograde consideration)
- **Naisargika Bala** - Natural strength
- **Drik Bala** - Aspectual strength

Include Rupas and percentile rankings for each planet.

## Technical Considerations

- All calculations should use the Swiss Ephemeris for maximum precision
- Implement efficient caching for repeated calculations
- Use coroutines for background computation of complex features
- Maintain offline-first architecture - no internet required
- Support for different Ayanamsa options (Lahiri, Raman, KP, etc.)
- Consider memory optimization for lower-end devices
- Implement proper error handling for edge cases (extreme latitudes, etc.)
